# 🚀 Jupiter Arbitrage Bot

**Автоматический арбитражный бот для поиска ценовых расхождений между централизованными (CEX) и децентрализованными (DEX) биржами с уведомлениями через Telegram.**

Бот анализирует ордербуки множества CEX и сравнивает их с котировками Jupiter (Solana DEX), используя математическую модель AMM-свопов для расчёта оптимальных объёмов сделок и прогнозирования прибыли.

## ✨ Ключевые возможности

- **Двунаправленный арбитраж** — поиск возможностей CEX→DEX и DEX→CEX
- **AMM Swap Model** — математическое моделирование DEX свопов через `curve_fit` (scipy) для точного прогнозирования выходных сумм без реальных запросов
- **Мульти-биржевой мониторинг** — одновременное отслеживание 9 CEX через WebSocket-подключения для ордербуков в реальном времени
- **Верификация сигналов** — двухэтапная проверка: сначала предсказание по модели, затем верификация через реальный запрос к Jupiter API
- **Учёт комиссий и сетей** — автоматическая проверка совпадения сетей (Solana), статусов депозита/вывода, и расчёт комиссий бирж
- **Stale Data Protection** — механизм кеширования данных о сетях/комиссиях с graceful degradation при сбоях API
- **Telegram-бот** — полнофункциональный бот с интерактивным интерфейсом для настройки фильтров и получения сигналов
- **PostgreSQL** — хранение пользовательских настроек и конфигураций
- **Многопроцессность** — каждый компонент работает в отдельном процессе с graceful shutdown

## ⚙️ Установка

### Требования

- **Python** 3.10+
- **PostgreSQL** (для Telegram-бота)
- Стабильное интернет-соединение

### Шаги

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Nur1k100/JupiterArbitrageBot.git
   cd JupiterArbitrageBot
   ```

2. **Создайте виртуальное окружение:**
   ```bash
   python -m venv .venv
   # Windows
   .venv\Scripts\activate
   # Linux/macOS
   source .venv/bin/activate
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

   Основные зависимости:
   | Пакет        | Назначение                              |
   |--------------|-----------------------------------------|
   | `ccxt`       | Унифицированный API для бирж            |
   | `aiohttp`    | Асинхронные HTTP-запросы                |
   | `websockets` | WebSocket соединения                    |
   | `aiogram`    | Telegram Bot API (v3)                   |
   | `psycopg2`   | PostgreSQL драйвер                      |
   | `numpy`      | Числовые вычисления                     |
   | `scipy`      | Оптимизация (curve_fit для AMM модели)  |
   | `pycryptodome` | Криптография (подпись LBank API)      |

4. **Настройте PostgreSQL:**
   ```sql
   CREATE DATABASE postgres;  -- или ваше имя БД
   ```
   Таблицы создадутся автоматически при первом запуске бота.

## 🔧 Конфигурация

### Файл `Configs/config.py`

Основные параметры:

| Параметр           | Описание                                  | Значение по умолч. |
|--------------------|--------------------------------------------|---------------------|
| `MINIMAL_PROFIT`   | Мин. прибыль для отправки сигнала (USDT)  | `20`                |
| `MINIMAL_PRED_PROFIT` | Мин. предсказанная прибыль (USDT)      | `10`                |
| `MINIMAL_DIFF`     | Мин. процентная разница цен               | `0.005` (0.5%)      |
| `MAXIMAL_DIFF`     | Макс. процентная разница цен              | `0.5` (50%)         |
| `PROXY`            | HTTP прокси для запросов                  | —                   |
| `API_TOKEN`        | Токен Telegram-бота (Config)              | —                   |

### API ключи бирж

API ключи для бирж указываются в файле `CEX/cex_network_fetcher.py` в словаре `API_CREDENTIALS`.

> ⚠️ **Безопасность:** Никогда не коммитьте API ключи в репозиторий. Используйте переменные окружения или `.env` файл.

### Telegram Bot

Токены Telegram-ботов настраиваются в:
- `Configs/config.py` → `API_TOKEN` (бот конфигурации)
- `All_services/telegramBot.py` → `BOT_TOKEN` (бот для сигналов)

## 🚀 Запуск

Система состоит из трёх независимых сервисов. Запустите каждый в отдельном терминале:

### 1. CEX Runner — сбор ордербуков и данных о сетях
```bash
python main_cex_runner.py
```
Запускает 9 CEX WebSocket клиентов + Network Fetcher в отдельных процессах.

### 2. DEX Runner — расчёт параметров AMM модели
```bash
python main_dex_runner.py
```
Запускает SwapModelCalculator и TokensFetcher.

### 3. Service Runner — сканер арбитража и Telegram-бот
```bash
python main_service_runner.py
```
Запускает Scanner, Telegram Bot и MSS WebSocket сервер.

### Порядок запуска

```
1. main_cex_runner.py     → Собирает ордербуки и данные о сетях
2. main_dex_runner.py     → Рассчитывает AMM параметры
3. main_service_runner.py → Ищет арбитраж и отправляет уведомления
```

> 💡 **Важно:** CEX и DEX раннеры должны быть запущены первыми, чтобы Scanner имел актуальные данные.

## 🧠 Как это работает

### 1. Сбор данных (CEX Runner)
Каждый CEX-модуль подключается к бирже через WebSocket и непрерывно собирает ордербуки (asks/bids) для всех пар `*/USDT`. Данные сохраняются в JSON-файлы в `CEX/orderbooks/`.

Параллельно **Network Fetcher** через API бирж (ccxt + нативные API) получает информацию о сетях: статусы депозита/вывода, комиссии за вывод.

### 2. Моделирование DEX (DEX Runner)
**SwapModelCalculator** использует формулу AMM (Automated Market Maker):

```
y_out = (Y_eff × x_in) / (X_eff + x_in)
```

Для каждого токена на Jupiter делается несколько запросов с разными объёмами, затем через `curve_fit` (scipy) определяются параметры `X_eff` и `Y_eff`, описывающие эффективную ликвидность пула. Это позволяет прогнозировать выходную сумму свопа для любого объёма без реального запроса к API.

### 3. Поиск арбитража (Scanner)

#### CEX→DEX (купить на бирже, продать на Jupiter)
1. Сравнение цены ask (CEX) с ценой bid (DEX модель)
2. Если разница > `MINIMAL_DIFF` — детальный анализ ордербука
3. Итеративный поиск оптимального объёма, максимизирующего прибыль
4. Проверка совпадения сетей (Solana) и статусов вывода
5. Верификация через реальный запрос к Jupiter API
6. Если прибыль > `MINIMAL_PROFIT` — отправка сигнала

#### DEX→CEX (купить на Jupiter, продать на бирже)
1. Сравнение цены bid (CEX) с ценой ask (DEX модель)
2. Аналогичный алгоритм с обратной моделью свопа
3. Верификация и фильтрация

### 4. Доставка сигналов
Сигналы передаются через **WebSocket** (MSS Server) в Telegram-бот, который фильтрует их согласно настройкам каждого пользователя и рассылает через персональные боты.
